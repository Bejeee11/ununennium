## Reporting Bugs

### Before Reporting

1. Check existing [Issues](https://github.com/olaflaitinen/ununennium/issues) for duplicates
2. Verify the issue with the latest version
3. Review the [Troubleshooting Guide](docs/guides/troubleshooting.md)

### Bug Report Template

When reporting a bug, include:

```markdown
**Environment**
- Ununennium version:
- Python version:
- PyTorch version:
- Operating system:
- GPU (if applicable):

**Description**
A clear description of the bug.

**Steps to Reproduce**
1. Step one
2. Step two
3. ...

**Expected Behavior**
What you expected to happen.

**Actual Behavior**
What actually happened.

**Code Sample**
```python
# Minimal code to reproduce the issue
```

**Error Messages**
Full traceback if applicable.

**Additional Context**
Any other relevant information.
```

## Feature Requests

Feature requests are welcome via [GitHub Discussions](https://github.com/olaflaitinen/ununennium/discussions).

### Feature Request Guidelines

| Aspect | Description |
|--------|-------------|
| Use case | Describe the problem you are trying to solve |
| Proposal | Suggest a potential solution |
| Alternatives | Note any alternatives you have considered |
| Context | Provide relevant background information |

## Security Vulnerabilities

For security issues, please follow the process in [SECURITY.md](SECURITY.md). Do not open public issues for security vulnerabilities.

---

## Response Times

We strive to respond to support requests in a timely manner:

| Request Type | Target Response |
|--------------|-----------------|
| Security vulnerabilities | 48 hours |
| Bug reports | 7 days |
| Feature requests | 14 days |
| General questions | Best effort |

> **Note**: Response times are targets, not guarantees. This is a community project maintained by volunteers.
